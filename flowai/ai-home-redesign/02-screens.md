# 02-screens — 플로우 AI 첫 화면

## 0. 공통

### 0-1. 상태 정의

| 상태 | 정의 |
|---|---|
| 자유대화 | 기능 미선택. 모든 기능 칩 노출. 예시 프롬프트 미노출 |
| 입력형 기능 선택 | 입력형 기능칩 1개 선택됨. 입력영역 내부에 기능 아이콘과 X 버튼 노출. 예시 프롬프트 노출 |

첫 화면 최초 진입 시 기본 상태는 `자유대화`다.

### 0-2. 화면 요소 역할 분리

| 요소 | 역할 |
|---|---|
| 상단 공지 바 | 짧고 중요한 공지 |
| 예시 프롬프트 | 입력 유도 |
| 슬라이드형 배너 | 상세 안내 및 행동 유도 |

세 요소의 메시지는 중복되지 않게 운영한다.

## 1. 첫 화면 구조

첫 화면은 다음 순서로 구성된다. 1순위 행동은 항상 `질문 입력`이다.

```
┌──────────────────────────────────┐
│ 상단 공지 바                       │
├──────────────────────────────────┤
│ 인사말                             │
│ 기능 선택 칩                        │
│ 기능별 예시 프롬프트 (조건부 노출)     │
│ 입력창                             │
│ 입력창 하단 슬라이드형 배너           │
└──────────────────────────────────┘
```

즐겨찾기 섹션은 첫 화면에서 제거한다 (§6 참조).

## 2. 기능 선택 칩

### 2-1. 칩 종류와 분류

| 칩 | 분류 | 동작 |
|---|---|---|
| 플로우에서 검색하기 | 입력형 | 첫 화면 입력 흐름 진입 |
| 웹에서 검색하기 | 입력형 | 첫 화면 입력 흐름 진입 |
| 이미지 만들기 | 입력형 | 첫 화면 입력 흐름 진입 |
| AI 챗봇 | 이동형 | 챗봇 홈 화면으로 이동 |

### 2-2. 자유대화 상태 동작

- 모든 기능 칩 노출.
- 예시 프롬프트 미노출.
- 입력영역에 기능 아이콘이 표시되지 않는다.

### 2-3. 입력형 기능칩 선택 동작

- 입력영역 내부에 선택 기능의 아이콘과 X 버튼 노출.
- 예시 프롬프트 노출 (§3).
- 입력창 placeholder가 해당 기능에 맞는 문구로 변경 (§4-1).

### 2-4. AI 챗봇 칩 동작

- 첫 화면 내 입력 흐름에 진입하지 않는다.
- 클릭 즉시 챗봇 홈 화면으로 이동.

## 3. 기능별 예시 프롬프트

### 3-1. 노출 조건

| 상태 | 노출 |
|---|---|
| 자유대화 | 미노출 |
| 입력형 기능 선택 | 해당 기능의 예시 프롬프트 세트 노출 |

각 입력형 기능별 2~4개의 예시 프롬프트를 제공한다. 예시 프롬프트는 사용자가 바로 활용 가능한 요청문 형태여야 한다.

### 3-2. 클릭 동작

| 동작 | 결과 |
|---|---|
| 예시 프롬프트 클릭 | 입력창에 문구 채움 |
| 자동 전송 | 적용하지 않음 |

## 4. 입력창

입력창은 첫 화면의 핵심 인터랙션 요소다.

### 4-1. placeholder 동작

| 상태 | placeholder |
|---|---|
| 자유대화 | 기본 문구 |
| 입력형 기능 선택 | 해당 기능에 맞는 문구로 변경 |

### 4-2. 기능 해제 (X 버튼)

| 항목 | 동작 |
|---|---|
| 해제 트리거 | 입력영역 내부의 X 버튼 |
| 상태 전환 | 자유대화 상태로 복귀 |
| 기능 칩 | 모든 기능 칩 재노출 |
| 예시 프롬프트 | 미노출 전환 |
| 입력 텍스트 | 작성된 텍스트 유지 |

## 5. 슬라이드형 배너

### 5-1. 위치·구조

| 항목 | 스펙 |
|---|---|
| 위치 | 입력창 하단 |
| 구성 요소 | 제목, 설명, CTA |
| 높이 | 고정 (화면 점프 방지) |
| 첫 노출 | 우선순위가 가장 높은 배너 |

배너 운영(백오피스 등록, 필수 입력값, 종료 처리)은 [01-PRD §9-3](01-PRD.md#9-3-배너-운영-정책-백오피스) 참조.
배너 노출 정책(우선순위, 최대 개수)은 [01-PRD §9-2](01-PRD.md#9-2-배너-노출-정책) 참조.

### 5-2. 자동 롤링

자동 롤링을 적용한다. 세부 운영 기준(속도·수동 조작 후 지속·hover 정지·접근성 대응)은 미결.

| 미결 항목 | 참조 |
|---|---|
| 자동 롤링 속도 | [HOME-001](06-open-issues.md#home-001) |
| 수동 조작 후 자동 롤링 지속 여부 | [HOME-002](06-open-issues.md#home-002) |
| hover 시 자동 롤링 정지 여부 | [HOME-003](06-open-issues.md#home-003) |
| 접근성 대응 기준 | [HOME-004](06-open-issues.md#home-004) |

## 6. 즐겨찾기

| 항목 | 처리 |
|---|---|
| 첫 화면 표시 | 제거 |
| 기능 자체 | 유지 |
| 저장된 즐겨찾기 데이터 | 유지 |
| 이관 영역 | 첫 화면 외 별도 영역 (상세 설계는 별도 기획) |

## 7. UX 원칙

- 첫 화면의 핵심은 `바로 질문할 수 있는 상태`를 만드는 것이다.
- 사용자는 기능 선택 직후 "이 기능으로 무엇을 할 수 있는지"를 즉시 이해할 수 있어야 한다.
- 예시 프롬프트는 입력을 돕는 장치이며, 선택 피로를 유발하지 않는다.
- 배너는 운영 메시지를 전달하되 입력 시작 흐름을 방해하지 않는다.
- 공지 바·예시 프롬프트·배너는 역할이 다르다 (§0-2).
