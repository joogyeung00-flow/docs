# 기능 개발 워크플로우: $ARGUMENTS

## 지시사항

아래 단계를 순서대로 실행하세요. 각 단계는 해당 서브에이전트에게 위임합니다.

### Step 1: PM - 기획 (서브에이전트: @agent-pm)
1. `$ARGUMENTS`에 대한 기능 스펙 문서를 작성합니다
2. 유저 스토리와 수용 기준(Acceptance Criteria)을 정의합니다
3. 디자이너를 위한 핸드오프 문서를 `docs/pm/handoff/`에 생성합니다
4. 완료 후 결과 요약을 반환합니다

### Step 2: Designer - 설계 (서브에이전트: @agent-designer)
1. PM의 핸드오프 문서(`docs/pm/handoff/`)를 읽습니다
2. 화면설계서를 작성합니다 (상태별 UI, 인터랙션 포함)
3. 필요한 컴포넌트 스펙을 정의합니다
4. `docs/design/screens/`에 산출물을 저장합니다
5. 완료 후 결과 요약을 반환합니다

### Step 3: Developer - 기술 설계 (서브에이전트: @agent-dev)
1. PM의 기능 스펙과 Designer의 화면설계서를 읽습니다
2. 기술 설계 문서(TDD)를 작성합니다
3. 구현 계획과 컴포넌트 분해를 정리합니다
4. 기획/디자인에 대한 피드백이 있으면 `docs/dev/feedback/`에 기록합니다
5. 완료 후 결과 요약을 반환합니다

### Step 4: 최종 요약
모든 에이전트의 작업이 완료되면, 전체 결과를 요약합니다:
- 생성된 문서 목록
- 각 에이전트의 주요 산출물
- 발견된 이슈나 피드백
- 다음 액션 아이템
