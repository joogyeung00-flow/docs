# 개발자 핸드오프: AI 스마트 비서 프로토타입 (FLOW-001)

---

## 1. 프로토타입 기술 스펙

### 1-1. 기술 스택

| 항목 | 선택 |
|------|------|
| 구현 방식 | 단일 HTML 파일 |
| CSS | Tailwind CSS (CDN) |
| JS | Vanilla JavaScript (ES6+) |
| 폰트 | Google Fonts - Noto Sans KR |
| 빌드 | 없음 (파일 열기만으로 실행) |

### 1-2. 기준 해상도
- 모바일 퍼스트: 390px 기준
- 데스크톱: 중앙 정렬 max-width 컨테이너

---

## 2. 구현 우선순위

### P0 (필수 구현)
1. 온보딩 Step1: 비서 이름 입력 + 유효성 검사
2. 온보딩 Step2: 키워드 칩 선택 UI
3. 온보딩 로딩 → 완료 전환
4. 브리핑 메인 화면: 4섹션 카드 렌더링
5. [내용 브리핑] CTA 클릭 → AI 답변 버블 추가

### P1 (가능하면 구현)
6. 자유 대화 입력 → 사용자/AI 메시지 교환
7. 설정 화면 (이름 변경, 말투 선택)
8. 화면 전환 애니메이션

### P2 (선택)
9. 로컬 스토리지 저장 (온보딩 상태 유지)
10. 설정 변경 시 브리핑 말투 반영

## 4. 인터랙션 구현 가이드

### 4-1. 온보딩 플로우

```
[Step 1: 이름 입력]
├── 입력 시: 실시간 글자 수 카운트 (n/10)
├── 금지어: 인라인 에러 메시지 표시
├── [다음] 클릭: 빈 입력 시 "플레인" 자동 적용 → Step 2로 전환
└── 전환: fade 또는 slide 애니메이션 (200ms)

[Step 2: 키워드 선택]
├── 칩 탭: 같은 카테고리 내 단일 선택 (토글)
├── 활성화 조건: 각 카테고리 최소 1개 선택
├── [아바타 생성하기] 클릭: Step 3 로딩으로 전환
└── [이전] 클릭: Step 1로 복귀 (입력값 유지)

[Step 3: 로딩]
├── 2초 setTimeout
└── 자동 전환: Step 4 완료 화면

[Step 4: 완료]
├── 축하 애니메이션 재생 (CSS keyframes)
├── 비서 이름 + 인사말 표시
└── [시작하기] 클릭: 브리핑 화면으로 전환
```

### 4-2. 브리핑 인터랙션

```
[내용 브리핑] 버튼 클릭:
1. 버튼 비활성화 (중복 클릭 방지)
2. 타이핑 인디케이터 표시 (점 3개 애니메이션, 1초)
3. AI 답변 말풍선 fade-in 추가
4. 채팅 영역 자동 스크롤 (최하단)

[메시지 전송]:
1. 입력값 가져오기 + 입력창 초기화
2. 사용자 말풍선 추가 (우측)
3. 타이핑 인디케이터 표시 (1.5초)
4. AI 답변 말풍선 추가 (좌측)
5. 자동 스크롤
```

## 5. 비기능 요구사항

| 항목 | 요구사항 |
|------|---------|
| 반응형 | 390px (모바일) ~ 1440px (데스크톱) |
