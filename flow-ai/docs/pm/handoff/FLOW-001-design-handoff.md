## 2. 화면별 UX 요구사항

### 2-1. 온보딩 플로우 (SCR-001 ~ SCR-006)

**설계 원칙**
- 전체 소요시간: 30초 이내 완료 가능해야 함
- 최소한의 입력으로 최대의 개인화 효과
- 이전 단계로 돌아갈 수 있어야 함 (뒤로가기)
- 진행 상태 표시 (스텝 인디케이터)

**SCR-001: 온보딩 진입**
- 목적: AI 비서의 가치를 한눈에 전달하고 시작 유도
- 요소: AI 비서 일러스트/아이콘(중앙), 소개 문구, [시작하기] CTA 버튼, "나중에 설정" 텍스트 링크
- 톤: 친근하고 기대감을 주는 문구

**SCR-002: 비서 이름 설정**
- 목적: 사용자가 AI 비서에 이름을 부여하여 애착 형성
- 요소: 텍스트 입력 필드 (최대 10자), 실시간 유효성 메시지 영역, 기본값 "플레인" 표시, [다음] 버튼
- 인터랙션:
  - 입력 시 실시간 글자 수 카운트 (n/10)
  - 금지어 입력 시 즉시 인라인 에러 메시지
  - 빈 입력으로 [다음] 탭 시 기본값 "플레인" 자동 적용
- 에러 상태: "특수문자는 사용할 수 없어요", "이 이름은 사용할 수 없어요"

**SCR-003: 아바타 키워드 선택**
- 목적: 최소 입력으로 아바타 개인화
- 요소:
  - 카테고리 1 - 분위기: 차분한 / 활기찬 / 따뜻한 / 시크한 (칩 UI, 단일 선택)
  - 카테고리 2 - 스타일: 심플 / 귀여운 / 전문적인 / 위트있는 (칩 UI, 단일 선택)
  - [아바타 생성하기] CTA 버튼
- 인터랙션: 칩 탭 시 선택 상태 토글, 선택된 칩 강조 표시, 최소 1개씩 선택해야 버튼 활성화

**SCR-004: 아바타 생성 로딩**
- 목적: 생성 대기 시간 동안 기대감 유지
- 요소: 로딩 애니메이션 (펄스 또는 스피너), "아바타를 만들고 있어요..." 텍스트
- 지속시간: 3~5초 (프로토타입에서는 2초로 단축)

**SCR-005: 아바타 확인**
- 목적: 생성된 아바타 확인 및 선택
- 요소: 아바타 이미지 크게 표시 (160x160), 비서 이름 표시, [확인] 기본 CTA, [다시 만들기] 보조 버튼
- 인터랙션: [다시 만들기] → SCR-003으로 돌아감

**SCR-006: 온보딩 완료**
- 목적: 완료 축하 및 첫 대화 유도
- 요소: 축하 애니메이션 (파티클/컨페티), 비서 아바타 + "만나서 반가워요! {이름}이에요" 인사말, [시작하기] CTA
- 자동 전환: [시작하기] 탭 → SCR-007 브리핑 화면으로 이동

### 2-2. 브리핑 화면 (SCR-007 ~ SCR-009)

**SCR-007: 브리핑 채팅 세션**
- 목적: 아침 브리핑을 대화형 UI로 전달
- 레이아웃:
  - 상단 헤더: AI 비서 아바타(좌측) + 이름 + 날짜(중앙) + 설정 아이콘(우측)
  - 중앙: 스크롤 가능한 채팅 영역
  - 하단 고정: 입력창 + 전송 버튼
- 브리핑 메시지 구조 (AI 말풍선):
  - 인사말: "좋은 아침이에요! 오늘의 브리핑이에요 ☀️"
  - 섹션 1 - 오늘의 일정: 일정 카드 최대 3건, 각각 [일정 브리핑] CTA
  - 섹션 2 - 마감 업무: 업무 카드 최대 5건, 우선순위 배지, 각각 [업무 브리핑] CTA
  - 섹션 3 - 부재중 변화: 변경사항 요약 카드
  - 섹션 4 - 확인 필요: 미회신 항목, 각각 [내용 브리핑] CTA
- 빈 상태: 각 섹션별 빈 상태 메시지 제공

**SCR-008: 내용 브리핑 응답**
- 목적: 특정 항목의 AI 요약을 대화 흐름으로 전달
- 트리거: [내용 브리핑] 버튼 클릭
- 동작: 사용자 말풍선 없이 AI 말풍선이 추가됨
- AI 답변 포맷: 자연어 요약 + 관련 정보 카드 (선택적)

**SCR-009: 자유 대화**
- 목적: 브리핑 맥락을 기반으로 추가 질문
- 동작: 입력창에 텍스트 입력 → 전송 → 사용자 말풍선(우측) → AI 답변 말풍선(좌측)

### 2-3. 설정 화면 (SCR-010)

**SCR-010: AI 비서 설정**
- 진입점: 브리핑 화면 상단 헤더의 설정 아이콘
- 전환: 우측에서 슬라이드인 패널 또는 별도 화면
- 설정 항목:
  - 비서 이름 변경: 텍스트 입력 (온보딩과 동일 유효성 규칙)
  - 아바타 재생성: [아바타 변경하기] 버튼 → SCR-003 플로우 재진입
  - 말투 선택: 격식체 / 반말체 / 귀여운체 (라디오 버튼)

## 3. 컴포넌트 요구사항

### 3-1. 필요한 컴포넌트 목록

| 컴포넌트 | 사용 화면 | 설명 |
|---------|----------|------|
| AI 아바타 | 전체 | 원형 아바타, 그라데이션 배경 + 이니셜/이모지 |
| 채팅 버블 (AI) | SCR-007~009 | 좌측 정렬, 회색 배경, 아바타 포함 |
| 채팅 버블 (사용자) | SCR-009 | 우측 정렬, 브랜드 컬러 배경 |
| 브리핑 섹션 카드 | SCR-007 | 섹션 제목 + 항목 리스트 + CTA 버튼 |
| 업무 항목 행 | SCR-007 | 업무명 + 우선순위 배지 + 마감 라벨 |
| 일정 항목 행 | SCR-007 | 시간 + 제목 + 장소 + 참석자 수 |
| 우선순위 배지 | SCR-007 | 긴급(빨강), 높음(주황), 보통(파랑) |
| CTA 버튼 | SCR-007~008 | 아웃라인 스타일, "내용 브리핑" 텍스트 |
| 키워드 선택 칩 | SCR-003 | 탭 가능, 선택/미선택 상태 구분 |
| 텍스트 입력 필드 | SCR-002, 009 | 플레이스홀더, 유효성 메시지 영역 |
| 스텝 인디케이터 | SCR-001~005 | 현재 단계 표시 (점 또는 바) |
| 설정 아이콘 버튼 | SCR-007 | 헤더 우측, 기어 아이콘 |

### 3-2. 상태 정의

- 버튼: default / hover / active / disabled
- 칩: unselected / selected
- 입력필드: empty / focused / filled / error
- 채팅 버블: 등장 시 fade-in 애니메이션

## 4. 디자인 가이드라인 요청

### 4-1. 브랜드 방향
- **전체 톤**: 전문적이지만 친근한 (corporate-friendly)
- **컬러**: 인디고 계열 브랜드 컬러 (#5B5BD6 제안)
- **배경**: 다크 모드 우선 (slate-900 계열)
- **여백**: 넉넉한 여백으로 가독성 확보

### 4-2. 디자인 토큰 요청 사항
- 컬러 팔레트 (브랜드, 시멘틱, 우선순위)
- 타이포그래피 스케일 (heading, body, caption)
- 스페이싱 시스템 (4px 기반)
- 보더 반경 (카드, 버튼, 칩, 아바타)
- 그림자/엘리베이션
